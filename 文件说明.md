## `src/` — 源代码

| 文件 | 角色 | 说明 |
|------|:----:|------|
| **`ocr_workflow_accelerated.py`** | ★ **PC 主程序** | 最终整合版，全 ONNX Runtime 直推 + Cls/Rec 批量推理。提供 6 个对外接口（`recognize_image` / `recognize_image_full` / `init_ocr` / `set_ocr_callback` / `batch_recognize` / `get_ocr_status`） |
| **`ocr_workflow_final.py`** | ★ **PC 主程序备份** | 内容同上，保留作为备份入口 |
| **`ocr_board_v3.py`** | ★ **板端最终版** | 全 ORT-CPU 推理（弃用 DLC-GPU），四核多线程优化，板端精度 ~0.949 / 88 框 |
| `cloud_ocrv5.py` | 云端 OCR | 调用云端 OCR API (`/ocr`)，批量识别 photos/ 图片 |
| `Paddleocr_VL_cloud.py` | 云端 Layout | 调用云端 Layout-Parsing API (`/layout-parsing`) |
| `camera_test.py` | 摄像头拍摄 | USB 摄像头 (IMX577) 拍照保存，仅设分辨率不碰硬件参数 |
| `camera_test_improve.py` | 摄像头增强版 | 自适应 Gamma 校正 + CLAHE + 去噪声预处理 |
| `asr.py` | 语音模块 | 异响检测 + 语音唤醒 + ASR 识别（独立模块，与 OCR 并行） |

### 各文件模型路径说明

`src/` 下的 OCR 程序路径已自动适配，模型存放在 `pp-ocrv4_rapid_onnx/`（相对于项目根目录），移动到 `src/` 后内部使用 `../pp-ocrv4_rapid_onnx/` 引用模型，**无需手动改路径**。

## 模型目录 `pp-ocrv4_rapid_onnx/`

**4 个核心文件（必须携带，共 ~15.3MB）**：

| 文件 | 大小 | 作用 |
|------|:----:|------|
| `ch_PP-OCRv4_det_mobile.onnx` | 4.5 MB | 文本检测 (Det) — 定位所有文字框 |
| `ch_ppocr_mobile_v2.0_cls_mobile.onnx` | 0.5 MB | 方向分类 (Cls) — 判断文字旋转角度 |
| `ch_PP-OCRv4_rec_mobile.onnx` | 10.3 MB | 文字识别 (Rec, FP32) — CTC 解码出文字 |
| `ppocr_keys_v1.txt` | 30 KB | 中文字典 (6623 字符) |

**可选文件**：
| 文件 | 说明 |
|------|------|
| `ch_PP-OCRv4_rec_mobile_int8.onnx` | 官方 INT8 量化版（实验性，板端精度崩坏） |
| `ch_PP-OCRv4_rec_mobile_custom_int8.onnx` | 自量化 INT8（PC QDQ 量化，CPU 不加速） |
| `*.dlc` | 骁龙 NPU 格式（旧版探索产物，暂未使用） |

## 快速使用指南

```bash
# 1. 单张图片 OCR 识别
cd cv_pc/src
python ocr_workflow_accelerated.py

# 2. 摄像头拍照 + OCR
python camera_test.py              # 拍照保存
python ocr_workflow_accelerated.py  # 识别照片

# 3. 云端 OCR（需要网络）
python cloud_ocrv5.py

# 4. 从其他代码调用 OCR
from src.ocr_workflow_accelerated import recognize_image
text = recognize_image("photos/photo_1.jpg")
```

> **依赖**: `pip install onnxruntime opencv-python numpy pyclipper shapely scikit-learn`
